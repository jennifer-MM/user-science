Usability analysis

Accessibility

Posted by Jennifer Menard on May 10, 2020

The role of digital content in our lives continues to grow. The number of organizations and companies transitioning a portion or all of their products and information is increasing. Most people file taxes online, register their children for school in the local school district, research purchases or complete school work. Having an internet connection and the ability to see or listen to the displayed is no longer an option.

The Web Content Accessibility Guidelines, WCAG, was developed as the global standard for web content accessibility to meet the needs of people with disabilities.

Executive Summary

This report describes how Johnson's Camera's website conforms with W3C's Web Content Accessibility Guidelines (WCAG).

Based on this evaluation, the website does meet WCAG 2.1, Conformance Level AA. Detailed review results are available in Section 6 below. Feedback on this evaluation is welcome.

Background and Evaluation

The results in this report are based on an evaluation conducted on the following dates: 7-10 May 2020. The website may have changed since that time.

Scope of Review

The website is a photographic studio, design house and specialty store. Three pages from the site were reviewed both manually and using WAVE, a semi-automated evaluation tool.

All the subpages of Johnson's Camera website were reviewed using WebAIMâ€™s WCAG 2 Checklist. It is a useful resource for technical implementation of WCAG. The criteria is:

Perceivable

  1. Content is made available to the sense- sight, hearing, and/or touch
  2. All linked images have descriptive alternative text
  3. Transcript or audio description is provided for non-live video
  4. Color is not used as the sole method of conveying content

Operable

  1. Provides manual controls for automatically moving, blinking, or scrolling content
  2. Don't design content in a way that is known to cause seizures or physical reactions
  3. Make it easier to use inputs other than keyboard
  4. Help users navigate, find content and determine where they are

Understandable

  1. Make text readable and easy to understand
  2. Make web pages appear and operate in predictable ways
  3. Help users avoid and correct mistakes

Robust

  1. Maximize compatibility with current and future assistive technologies

Results and Recommended actions

The WAVE web accessibility evaluation tool helps web developers make site features accessible for people with disabilities. It identifies actionable items to fix on the spot.

The WAVE tool organizes its report into the categories:

  1. Errors
  2. Contrast Errors
  3. Alerts
  4. Features
  5. Structural Elements
  6. ARIA
screen capture, www.johnsoncamera.com with WAVE tool screen capture, www.johnsoncamera.com/workshops with WAVE tool

The evaluation of Johnson's Camera website identified a number of errors. When we examine the details we learn that many errors prevent people with specific disabilities from accessing the content.

Perceivable

Findings Recommendations
  • Very low contrast
  • Missing alternative text
  • Missing form label
  • Increase contrast between foreground (text color) and background colors
  • Add an HTML alt attribute to image
  • Label the form control

Operable

Findings Recommendations
  • Multiple form labels
  • Only use one associated label element.

Understandable

Findings Recommendations
  • The page's language is not identified
  • Missing alternative text
  • Missing form label
  • Use HTML lang attribute to identify language
  • Add an HTML alt attribute to image
  • Label the form control

Robust

Findings Recommendations
  • All ARIA properties are present
  • Continuing using ARIA correctly

Conclusion

The overwhelming majority of Johnson's Camera website accessibility errors will be solved by adjusting the text and background colors. In this case, making the site accessible would take a reasonable amount of time and effort. The WAVE tool provides accurate information about how to fix challenges, quickly. All companies should include it in their accessibility testing process.
